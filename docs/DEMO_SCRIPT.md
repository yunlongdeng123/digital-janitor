# Digital Janitor - 3分钟演示脚本

**目标受众**：计算机课程评审老师、技术同行  
**演示时长**：3分钟  
**演示环境**：Windows 10 + Python 3.10 + Streamlit

---

## 🎬 开场（30秒）

### 台词

> 大家好！今天为您展示的是 **Digital Janitor** —— 一个基于大语言模型和 OCR 的智能文件归档系统。
>
> 先看一个真实痛点：（**切换到桌面**）这是我的下载文件夹，里面有几十个文件，文件名完全无意义：`扫描件0001.pdf`、`IMG_20241215.png`、`未命名.docx`... 每次找文件都要靠"最近修改时间"排序，效率极低。
>
> **Digital Janitor 要解决的问题就是：让 AI 自动识别这些文件，智能重命名并归档到合理的文件夹中。**

### 操作步骤

1. 打开下载文件夹，展示混乱的文件列表（3-5秒）
2. 快速滚动，突出"无意义文件名"（2秒）
3. 切换到项目目录（展示 `inbox/` 文件夹，里面有准备好的测试文件）

---

## 🤖 场景一：全自动归档 - OCR + 智能路由（60秒）

### 台词

> 首先演示 **CLI 模式 + 自动批准**。我们有3个测试文件：
>
> 1. 一张**发票扫描件**（完全看不出内容）
> 2. 一份**电子合同** PDF
> 3. 一张**产品截图**
>
> 我们运行命令：
>
> ```bash
> python run_graph_once.py --limit 3 --auto-approve --execute
> ```
>
> （**等待执行**）
>
> 看！系统正在处理：
>
> 1. **第一个文件**（扫描件）：触发了 **OCR 识别**，使用 RapidOCR 本地引擎，提取出"阿里云"、"1580元"、"2024-12-15"，LLM 判断为发票。**路由策略**选择了日期分层，归档到 `财务/2024/12/`，文件名变成了 `发票_2024-12-15_阿里云_1580元.pdf`。
>
> 2. **第二个文件**（合同）：电子版 PDF，直接提取，识别出供应商"腾讯科技"，使用**语义化路由**，归档到 `合同/腾讯科技/`。
>
> 3. **第三个文件**（截图）：图片类型，按日期归档到 `图片/2024/12/`。
>
> （**打开 archive 文件夹展示结果**）
>
> 可以看到，3个文件都被自动分类、重命名并移动到了合理的位置。**关键技术点：OCR 缓存**（第二次处理同一文件时，耗时从 2秒降到 0ms）。

### 操作步骤

1. 打开终端，`cd` 到项目目录（5秒）
2. 运行命令：`python run_graph_once.py --limit 3 --auto-approve --execute`（10秒）
3. **同步讲解**：指着终端输出，强调关键信息：
   - "触发 OCR"
   - "路由: 日期分层"
   - "已移动"
4. 打开 `archive/` 文件夹，展示生成的目录结构（10秒）
5. 打开其中一个文件，验证内容正确（5秒）

## 🤝 场景二：人工介入 - HITL 交互式审批（60秒）

### 台词

> 当然，不是所有文件都适合自动处理。比如这个扫描件 **OCR 质量很低**，或者 AI 的分类不太准确，这时就需要**人工介入**。
>
> 我们先生成待审批文件：
>
> ```bash
> python run_graph_once.py --limit 5
> ```
>
> （**切换到 Web UI**）
>
> 现在打开 Streamlit 审批界面（已经在后台运行）。可以看到**待审批队列**，展示了5个文件的详细信息：
>
> - **原始文件名** vs **AI 建议的新名字**
> - **LLM 分析理由**（为什么判断为发票）
> - **内容预览**（前500字符）
> - **元数据**（日期、金额、供应商）
>
> （**点击展开其中一个**）
>
> 这里有一个发票，AI 建议归档到 `财务/2024/12/`，但我希望把阿里云的发票单独放一个文件夹。我可以直接**修改目标目录**为 `财务/阿里云/2024/`，然后点击**批准**。
>
> （**操作：修改 → 批准 → 文件消失**）
>
> 文件立即被移动了。**重点来了**：系统会把这个操作记录到 **Memory 数据库**，学习到"阿里云的发票应该放在 `财务/阿里云/`"这个偏好。

### 操作步骤

1. 终端运行：`python run_graph_once.py --limit 5`（10秒）
2. 切换到浏览器（Streamlit 已在 `localhost:8501`）（5秒）
3. 展示待审批队列（5秒）
4. **点击展开**一个发票文件（5秒）
5. **修改目标目录**输入框（10秒）
6. **点击批准**（5秒）
7. **刷新**（文件消失，显示"已移动"提示）（5秒）
8. 快速展示侧边栏的"今日已处理"数字增加（5秒）

## 🧠 场景三：越用越聪明 - Memory 偏好生效（40秒）

### 台词

> 现在是最精彩的部分：**Memory 学习系统**。
>
> 我们再次运行归档命令，处理一个新的阿里云发票：
>
> ```bash
> python run_graph_once.py --limit 1 --auto-approve --execute
> ```
>
> （**观察输出**）
>
> 注意！这次输出显示：**"路由: Memory 偏好命中 → 财务/阿里云/2024/"**！
>
> AI 自动应用了我们上次的修改，不再使用默认的 `财务/2024/12/`，而是直接用了我们偏好的路径。这就是**增量学习算法**的效果。
>
> （**切换到 Web UI 的"🧠 学习到的偏好"页面**）
>
> 在这里可以看到，系统记录了这条规则：
>
> - **供应商**：阿里云
> - **文档类型**：invoice
> - **目标文件夹**：财务/阿里云/2024/
> - **置信度**：0.7（达到应用阈值）
> - **样本数**：2
>
> 也就是说，**下次再遇到阿里云的发票，系统会自动使用这个路径，不需要我再手动修改**。

### 操作步骤

1. 终端运行：`python run_graph_once.py --limit 1 --auto-approve --execute`（10秒）
2. **指着输出**："Memory 偏好命中"（5秒）
3. 切换到 Web UI，点击侧边栏"🧠 学习到的偏好"（5秒）
4. **展开**偏好卡片，展示置信度、样本数（10秒）
5. **回到 archive 文件夹**，验证文件确实在 `财务/阿里云/2024/` 下（5秒）
